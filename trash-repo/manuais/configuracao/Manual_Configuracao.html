Módulo Configuração



Este módulo tem por objetivo final permitir o controle de todos os outros módulos, controle de usuários, departamento, senhas, perfis de acesso, dicionário de dados do sistema, gerador de programas e outras funcionalidades que servem de base para todo o sistema. Controle das instituições e autarquias que estarão utilizando o sistema.

Cadastros



Itens de Menu



Esta rotina destina-se ao cadastro dos itens de menu do sistema. Um programa (uma página web) é identificada adicionando-se um item de menu e ligando este item de menu a um outro item de menu, formando assim um árvore de acesso.

Sempre que desejarmos cria um árvore de menus, como por exemplo: Configuração->Cadastro->Item de Menu, devemos para cada item a ser acessado criar um item de menu e depois na rotina de organizar itens de menu, ligar um ao outro para formarmos um menu de acesso.

Diferente dos outros programas, nesta opção o usuário poderá incluir um item, alterar ou excluir.

Segue orientação para o preenchimento dos campos conforme ilustrado na figura 01.


Descrição: Campo onde deve ser inserido a descrição do item, esta descrição será listada na opção do menu.

Ajuda: Colocar neste campo a mensagem de ajuda que aparecerá na barra de status (linha inferior) do DBPortal.

Função: Colocar o nome da página web que será chamada quando o item de menu for clicado.

Descrição Técnica: Colocar a descrição técnica do programa, sua finalidade e principais características.

Ambiente: Selecionar qual o ambiente que será gerado o menu. Esta opção destina a opção de estarmos criando dois ambientes de programação, podendo dividir os menus em duas estruturas. Opção utilizada na construção do sistema e em desuso.

Módulo: Selecionando esta opção, este item de menu será demonstrado como um módulo e não como um item de menu e neste caso, na tela de inclusão de itens este não aparecerá. Caso seja cadastrado como módulo, este será automaticamente cadastro um grupo de itens e devera ser adicionado na opção de organizar menus os itens, cadastro, relatório, consulta e procedimento.

Nome: Colocar o nome do módulo que será apresentado na tela de seleção dos módulos e outras rotinas que mostra.

Descrição: Colocar a descrição do módulo que será apresentado na tela de seleção dos módulos e outras rotinas que mostra..

Imagem: Colocar a imagem que será exibida na tela dos módulos.

Tem Exercício: Selecionar se neste módulo terá controle por exercício.

Liberado p/ Cliente: Controla se este item de menu esta liberado para os clientes ou não.

Código: Colocar o código do procedimento ao qual este item de menu será integrado. Este código identifica um procedimento, um conjunto de ações executadas no sistema para atender a um processo de trabalho.

Selecione um programa: Colocar todos os programas (páginas webs) que estarão ligadas ao item de menu. Este campo é obrigatório quando um programa é incluído no campo Função.

Inclusão


Para inclusão de um item de menu, basta preencher os campos desejados e clicar no botão de inclusão, nunca esquecendo que quando for um item que será utilizado para carregar um programa, os arquivos deverão estar incluídos na tela de “Selecione um Programa”.


Alteração


Para alterar um determinado item de menu, poderá ser selecionado o mesmo de duas formas:

a) Colocando a descrição do item de menu e depois clicando no botão “Procurar”, após o sistema mostrará uma lista com os menus que se enquadram na descrição digitada e clicando sobre qualquer um dos item listados no grid o sistema mostrará os dados do item de menu para alteração.

b) Clicando no botão “Seleciona”, o sistema mostra a lista dos módulos e se selecionado os módulos, mostrará a árvore dos itens de menus cadastrados para este módulo e clicando sobre o item o mesmo passa para a tela de alteração.


Exclusão


Para selecionar um item para a exclusão, segue-se os mesmos passos para alterar, onde além do botão alterar será liberarado também o botão excluir.


Acesso ao Sistema


Esta rotina destina-se ao cadastro das regras de acesso ao sistema, pessoas ou computadores autorizados. Este procedimento é uma parte do conjunto de opções de segurança que estão disponíveis no DBPortal.

Nesta opção o usuário administrador poderá cadastrar um acesso para um computador ou pessoa por um determinado período de tempo e hora ou tampouco por tempo indeterminado.

Inclusão


Código da Regra: Campo identifica o código da regra cadastrada.

Data Início: Data de início da validade da regra cadastrada.

Hora Início: Hora de início da validade da regra cadastrada.

Data Final: Data Final da validade da regra cadastrada.

Hora Final: Hora Final da validade da regra cadastrada.

Código do Usuário: Código do usuário que esta cadastrando a regra e nome do usuário.

Data Cadastro: Data do cadastro da regra.

Observação: Este campo destina-se ao registro das observações que ficarão registradas a cerca desta regra.


Após preencher os campos acima e clicar no botão de “Incluir”, o sistema libera as abas “Usuário”, “IP/Máscara”, “Cancela”.

Na aba de Usuário, o sistema solicita o código do usuário (login) que será verificado neste cadastro de acesso, isto é, quando o usuário acessar o sistema, esta regra será executada para ele.

Na aba de IP/Máscara, o sistema solicita o preenchimento da mascará que será verifica no momento do acesso ao sistema. Se colocado um asterisco “*” junto a máscara, o sistema entenderá qualquer código será aceito e não fará a restrição do acesso.

Na aba “Cancela” o usuário administrador poderá cancelar a regra de acesso cadastrada, colocando a descrição do porque que esta cancelando e incluindo o registro. Nos botões de alterar e excluir, poderá o mesmo cancelar o cancelamento da regra ou alterar a descrição nela informada.

No caso do preenchimento da aba de “Usuário” e “IP/Máscara” serem feitas simultaneamente, o sistema fará a consistência das duas chaves adicionadas a este regra quando do acesso ao sistema.

Uma regra somente é executada quando do acesso ao sistema. Caso seja cadastrada uma regra com o usuário ou computador já acessado ao sistema, esta será validada somente no próximo acesso, portanto deverá o administrador solicitar o encerramento da seção em questão.


Departamentos


Esta rotina destina-se ao cadastro dos departamentos. Um departamento identifica um setor dentro do órgão público, sendo utilizado na maioria dos módulos. Um exemplo claro da utilidade deste cadastro é no módulo de protocolo, quando do tramite dos processos, onde cada departamento tem como objetivo o agrupamento dos processos dentro do setor. As escolas, postos de saúde, unidades de almoxarifado são todos ligados ao cadastro dos departamentos.

Usuários


Usuário Interno



Considera-se usuário interno, o usuário que terá acesso ao DBPortal e portanto receberá permissões de acesso e outros.

Inclusão


A tela dos usuários internos exibe quatro abas, sendo “Usuários”, “Departamento”, “Liberar Perfis” e “Permissão de Empenho”.


Na aba “Usuários”, os campos abaixo deverão ser preenchidos.


Código Usuário: Código do Usuário.

Numcgm: Código do CGM – Cadastro Geral do Município. Clicando sobre a descrição (link), o sistema trás a uma tela com a lista de todos os nomes cadastrados no sistema.

Login do usuário: Login do usuário. O sistema sugere neste caso o login do usuário a ser incluído, colocando o primeiro nome como padrão. Sugere-se cadastrar os logins dos usuários com o primeiro nome, separado com ponto (.) e o sobrenome para possibilitar um número maior de logins e a identificação mais adequada.

Senha: Senha do Usuário.

Verifica senha: Confirmação da Senha do Usuário.

Usuário Ativo: Situação do Usuário.

Administrador: Usuário Administrador

Email: Email do usuário.

Instituição: Instituição que o Usuário terá acesso. A instituição que o usuário esta acessando será selecionada automaticamente.


Uma peculiaridade deste cadastro é a seleção das instituições e neste caso, uma ou mais instituições deverão ser cadastradas para um usuário. Estas instituições serão mostradas na tela de seleção das instituição ao acessar o sistema.

O usuário administrador será o usuário que terá acesso a maioria dos itens de menus, sem necessidade de liberar permissão para o mesmo. Este usuário poderá acessar o sistema mesmo que o mesmo esteja com inativo para os usuários.

Após o preenchimento destes campos e clicar no botão Incluir, o sistema libera as demais abas.

Na aba “Departamento”, poderá ser selecionado um ou mais departamentos para o usuário.

Na aba “Liberar Perfis” poderá também ser liberados um ou mais perfis para o usuário em questão ou também não efetuar nenhuma alteração.

Na aba “Permissões de Empenho” o sistema possibilita a atualização das permissões dos empenhos demonstrado no help desta opção.

Estas abas tem por objetivo a facilidade de manutenção, uma vez que não necessita de permissão, nem tão pouco o acesso as várias rotinas.

Os campos destas abas estão descritos no help de cada um dos itens de menu.

Figura 006

Alteração


Ao acessar a tela de alteração, uma lista de usuários internos será exibida e ao selecionar um dos usuários do sistema mostrará os dados como descrito na opção de inclusão.

Na alteração, após a seleção de um usuário o sistema libera as abas e coloca disponível o botão alterar. Também poderá ser alterado os dados de qualquer uma das abas, clicando sobre a descrição da aba e em seguida o sistema possibilitará a manipulação dos dados da mesma.

O campo Senha e Verifica Senha não deverão ser preenchidos, assim o sistema não modificará o seu conteúdo, pois nos procedimentos o sistema tem uma opção específica para esta ação.

Sugere-se que quando um usuário seja desligado da instituição, o mesmo deverá ser colocado como inativo e sua senha alterada para outra qualquer.

Perfil



Um Perfil é um identificador, um grupo onde pode-se colocar várias permissões e depois adicionar à vários usuários internos para facilitar o controle das permissões. Esta opção possibilita o controle de ativação e desativação dos itens de menu.

Um Perfil é basicamente um usuário, mas não possibilita o acesso ao sistema e outros exclusivos de usuário interno.

Inclusão


A tela de Perfis exibe os seguintes campos.


Código do Perfil: Código do Perfil

Nome do Perfil: Nome do Perfil, normalmente identificado com o nome da instituição, seguido da descrição resumida que possa identifica-lo.

Login: Login de identificação do perfil

O botão incluir executa a criação de um novo documento e libera a aba de parágrafos para a manutenção.

O botão Importar Documento, importa um documento já criado no sistema para este documento, importa também os parágrafos incluídos. Ao clicar neste botão o sistema abre uma tela de seleção onde estão listados todos os documentos/parágrafos incluídos e ao selecionar uma delas o sistema ainda solicita a confirmação de processamento ou não desta operação.

Já o botão Importa Documento Padrão, busca o documento no item de Documentos/Parágrafo Padrão e importa, criando o novo documento. Assim como o outro botão de Importar, o sistema também solicita que seja confirmado o processamento de importação das informações.

Na aba de parágrafos podemos utilizar os seguintes botões: Selecionar Parágrafo, Incluir, Parágrafo Novo, Ordenar e Visualizar.

Pressionando o botão Selecionar Parágrafos o sistema mostrará uma tela onde poderemos selecionar vários outros parágrafos e incluí-los de uma vez só, facilitando a seleção dos mesmos e a manutenção deste programa.

Para o botão incluír, primeiramente precisamos acessar o campo Código do Parágrafo, será mostrada uma tela onde selecionamos um parágrafo e este será mostrado neste campo e após pressiona-se o botão incluir para efetivar o processamento.

No botão Parágrafo Novo podemos criar um novo parágrafo, podemos dar um título, colocar alinhamento, distância em pixes do início da linha, espaçamento entre as linhas e o conteúdo a ser impresso. No campo conteúdo podemos utilizar algumas variáveis, entre em contato com o suporte mais verificação das mesmas.

No botão Ordenar podemos trocar a ordem dos parágrafos incluídos, a ordem que será impressa no documento. O sistema irá mostrar uma tela com os parágrafos e o usuário poderá movimentar clicando nas setas com indicação para cima e para baixo.

O botão visualizar, processa o documento gerado e gera uma impressão bem sucinta dos parágrafos digitados no documento.


Alteração

Esta opção assemelha-se no tratamento da opção inclusão, na aba de Documento possibilita-se que seja alterado as informações e utilizando-se do botão Alterar pode-se processa-las. O botão Pesquisar possibilita o acesso a tela que efetuamos a seleção no momento que acessamos este item.

Para a aba de Parágrafos, o procedimento de manipulação dos dados é o mesmo da opção de inclusão, não necessitando sua repetição.



Manutenção de Layouts



A manutenção de Layouts tem por objetivo organizar os mais diversos tipos de layouts gerados no sistema, leiouts de geração de dados para as instituições financeiras, caged, contra cheques e outros.


Inclusão


Este item possui três abas de manipulação dos dados, sendo: Layout, Linhas e Campos.


Na aba Layout temos a identificação:


Código do Layout: Código sequencial de identificação do layout

Descrição: Descrição do Layout

Observações: Observação do Layout para melhor identificação

Quantidade de Linhas: Numero padrão de linhas do Layout


O botão Incluir será exibido e o Importar Layout. O botão incluir processa a inclusão dos dados informados acima e o botão Importar Layout mostrará uma lista dos layout e se selecionado mostrará na tela os botões de Importar e Novo. O botão Importar nesta ação, processará a importação de todas as linhas e campos cadastrados no Layout selecionado e o botão Novo redefinirá a tela de inclusão como se acessado novamente o item de menu.

O funcionamento padrão das abas de Linhas e Campos se dá da seguinte forma, para cada linha que estamos editando, podemos adicionar, retirar os campos na aba campos, mas quando não temos linha selecionada o sistema não permite a manipulação da aba Campos.


Na aba linha temos os seguintes campos:


Código da linha: Código identificador da linha

Código do layout: Código do Layout da Linha

Descrição: Descrição da Linha

Tipo de linha: Tipo da linha que será geradas, sendo: Header de arquivo,Header de lote, Registro, Trailler de lote, Trailler de arquivo

Tamanho da linha: Tamanho da linha

Observações: Observações para identificar a linha

Quebras antes: Quantidades de quebras antes do campo

Quebras depois: Quantidades de quebras depois do campo


Para esta aba o sistema apresenta dois botões, Incluir e Importar Linha, sendo o Incluir a inclusão de uma linha nova e Importar Linha a seleção de uma linha já incluída em qualquer outro documento. Ao ser pressionado o botão Importar Linha o sistema mostra uma tela para seleção e em selecionado um linha mostra os botões com as seguintes descrição: Importar Linha, Importar Linha e Campos e novo.

No botão Importar Linha, o sistema somente copiará os dados da linha selecionada anteriormente e incluirá neste Layout. No botão Importar Linha e Campos importará além dos dados das linhas todos os campos a ela adicionados. Já o botão novo fará com que o programa seja iniciado com se acessado pela primeira vez.

Para esta abas, existe uma janela onde será mostrada todas as linhas cadastradas para este layout, podendo se clicar do A ou E alterar ou excluir uma linha digitada. Este procedimento facilita a digitação de linhas, bem como alteração de seus dados e exclusão de alguma linha digitada indevidamente.

Nome do campo: Nome do Campos

Temos nesta aba, o botão incluir e Importar Campo, sendo o incluir o processamento da inclusão dos dados no sistema e o Importar Campo a seleção de um campos já incluído, nesta ou em qualquer outra linha deste ou de qualquer outro documento. Mostra uma tela com os campos a Importar e se selecionado um, atualiza a tela com todos os dados do campos selecionado e pressionando-se novamente o botão Incluir, processamos então a inclusão dos mesmos no banco de dados.

Assim como na aba linha, nesta aba também temos na parte inferior uma janela contendo todos os campos cadastrados para a linha em manutenção e podemos acessando o link A e E proceder com as ações de manutenção, alterando ou excluindo um campo quando necessário. Em acessando um destes links o sistema irá personalizar o botão para proceder com a modificação na base de dados.


Alteração


A diferença da alteração para a inclusão é que na abas Documento pode-se alterar os dados do documento escolhido, como descrição por exemplo, mas, as abas linha e campos segue com o mesmo funcionamento da inclusão.


Exclusão


No caso da exclusão, o sistema ao ser acessado o item, mostra uma tela de seleção e após selecionado, mostra o botão Excluir e não libera as abas de linha e campos para visualização.

Ao selecionar um documento e pressionar o botão excluir o sistema eliminará todos os dados referente a este documento, suas linhas e campos.

Este procedimento é bastante utilizado quando do cálculo de IPTU para um novo exercício, onde gostaríamos de fazer algumas simulações cálculo, modificando parâmetros e outros sem comprometer a base de dados real e fiel do sistema.

Este item de menu não possui programa ligado ao mesmo, serve somente como identificador de permissão.

Status do Sistema



Esta opção destina-se ao controle de acesso ao sistema, possui três tipos de ações:

On Line: O sistema esta ativo e pode ser acessado por qualquer usuário que tenha login e senha e que possa acessar o mesmo.

Não Permite Novos Logs: O sistema em ativo, mas não permite que mais usuários acessem o mesmo. esta opção é válida quando necessitamos desativar o sistema mas temos usuários conectados. Então colocamos neste estado até saírem todos os usuários.

Off Line: O sistema está desativado e não permitirá nenhum tipo de acesso, não permitirá que nenhum usuário acesse nenhuma das páginas, significa que vai deslogar todos os usuários no próximo movimento que ele fizer no sistema.

Ativar e Desativar Trace Log



Ativar e desativar trace log, é uma opção usitlizada pela programação quando necessitamos verificar os procedimentos de consulta de um determinado programa. Esta opção é configurada por sessão de acesso, pois um usuário pode estar com esta opção ativada e outro não.

Configuração de Tela de Login



Esta opção destina a inclusão de tela de login padrão. Tela de login padrão é a tela inicial do sistema personalizada.

Configuração Inicial



Primeiro crie um arquivo com o nome padrão de login_<coplemento de identificação>.php ex : login_ecidade.php, login_dbseller.php, e salve na pasta templates no diretorio do e-cidade.

Configurar o arquivo require_extensions.xml da pasta config no diretório do e-cidade para redirecionar para o novo arquivo criado. Para isso devemos alterar os atributos .

<template_login label='Tela de Login do Sistema' src='templates/login_dbseller.php'></template_login>

ATENÇÃO: Não colocar o nome do arquivo como index.php por padrão nesse diretório já existe um arquivo com esse nome.


Requisitos



Após edite o arquivo criado para nova tela de login, devemos incluir os arquivos padrão de javascript :

<script language="JavaScript" type="text/javascript" src="scripts/md5.js"></script>

<script language="JavaScript" type="text/javascript" src="scripts/strings.js"></script>

<script language="JavaScript" type="text/javascript" src="scripts/prototype.js"></script>

Na tag <form> deve ser colocado o nome como form1 <form method="post" name="form1">.

Os elementos da tela (campos usuário, senha e botão de login) devem ter os atributos name e id conforme padrão que segue :

<input name="login" id='usu_login'>

<input name="senha" id='usu_senha' type="password">

<input name="btnlogar" type="button" class="botao" id="btnlogar" value="Acessar">

No arquivo login.php no diretório do e-cidade no conteúdo da tag <script></script>,

por padrão possui a função function js_addevent() que contem os eventos de cada <input > da tela de login.

Essa função deve ser chamada no evento onload da tag <body> no seu arquivo de login novo, <body onload="js_addevent()">.

Estrutura do Sistema

Cadastro de Módulos



Esta opção destina inclusão de módulos. Entende-se por módulos o conjunto de tabelas, como organizamos os dados dentro do banco de dados.

Cadastro de Tabelas



Esta opção destina inclusão de tabelas. Tabelas são a identificação do agrupamento de campos. Identificam um conjunto de tipos de dados diferentes ou não que identificam um pessoa, um objeto, etc.

Cadastro de Campos



Esta opção destina inclusão de campos. Campos são atributos que desejamos guardar de um determinado dado, por exemplo: nome de uma pessoa, descrição de um objeto, código, etc.

Consulta Estrutura do Sistema



Esta opção destina a consulta do dicionário de dados do sistema. Lista todos os módulos de controle de dados, todos os arquivos, campos das tabelas, índices dos campos, chaves estrangeiras, sequencias e outros dados pertinentes ao mesmo.

Organiza Campos



Esta opção destina à ligar todos os campos às tabelas, montar o dicionário de atributos e identificações que utilizaremos para compor o sistema.

Cadastro de Funções



Este cadastro documenta as funções que utilizamos no sistema, funções estas executadas no bando de dados.

Cadastro de Triggers



Este cadastro documenta as triggers, gatilhos que são processados quando da movimentação de informações nos campos dos arquivos do banco de dados. São utilizados normalmente para validação das informações inseridas entre os mais diversos cadastros do sistema.



Cadastro de Sequência



Este cadastro documenta as as sequências geradas para os campos, em sua grande maioria campos com contagem automática e que não podem se repetir.



Inclusão de Classe PHP Externa



Este cadastro documenta todas as classes de manipulação de dados e regras de negócio que utilizamos e que o gerador de programa não atende.

Gera Estrutura do Sistema



Esta rotina gera um arquivo contendo a estrutura do sistema.

Gerador de Programa



Esta rotina gera todos os arquivos necessários para manipulação dos dados do DBPortal, gera os itens de menu e outros dados.

Verifica Estrutura dos Arquivos



Esta rotina processa a validação das estruturas dos arquivos com a base de dados atual, para podermos validar se algo foi modificado na base e não foi alterado no dicionário de dados.

Consulta Campos



Esta rotina mostra a ligação entre os mais diversos campos de dados do sistema. Identifica quando um campos que identifica um determinado dado esta em um ou mais tabelas do sistema.
